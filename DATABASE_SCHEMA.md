# Cirlot Documents — Database Schema

All data is stored inside the standard WordPress database. No custom tables are created.

---

## 1. Custom Post Type — `cirlot_document`

**Table:** `wp_posts`

| Column | Value |
|---|---|
| `post_type` | `cirlot_document` |
| `post_title` | Document title |
| `post_status` | `publish` / `draft` / etc. |
| `post_date` | WordPress creation date (auto) |

---

## 2. Post Meta — `wp_postmeta`

One row per field per document (`post_id` = ID of the `cirlot_document` post).

| `meta_key` | Type | Description |
|---|---|---|
| `_document_file_id` | `int` | WordPress attachment ID of the uploaded file |
| `_document_pub_date` | `string` | Publication date in `YYYY-MM-DD` format |
| `_document_file_format` | `string` | One of: `pdf`, `word`, `excel` |
| `_document_description` | `text` | Value of the built-in Description field |
| `_document_cf_{field_id}` | `text` | Value of each custom field (one row per field). `{field_id}` matches the `id` defined in Settings → Custom Fields. Example: `_document_cf_resolution_number` |
| `_document_summary` | `text` | 1–2 sentence AI-generated summary (written by "Process with AI" or "Index for Search"). Used to enrich semantic indexing. |
| `_document_embedding` | `text` (JSON) | 768-dimension float vector as a JSON array (e.g. `[0.012, -0.034, …]`). Generated by the Gemini `text-embedding-004` model. Used for cosine-similarity search. |

### Notes
- `_document_description` is a special case: it maps to the custom field with `id = "description"`, but is stored under its own dedicated key instead of `_document_cf_description`.
- `_document_embedding` can be large (~18 KB per document). For catalogs under ~500 documents this is acceptable in `wp_postmeta`.
- `_document_summary` and `_document_embedding` are written automatically when "Process with AI" or "Index for Search" is used. They are never written on manual save.

---

## 3. Taxonomies

### `document_audience`
**Tables:** `wp_terms`, `wp_term_taxonomy`, `wp_term_relationships`

Stores audience labels (e.g. `Institution`, `Evaluator`, `Public`). Terms are created automatically when the taxonomy list is saved in Settings → Taxonomy.

### `document_type`
**Tables:** `wp_terms`, `wp_term_taxonomy`, `wp_term_relationships`

Stores document type labels (e.g. `Policies`, `Guidelines`, `Handbooks`). Terms are created automatically when the type list is saved in Settings → Taxonomy.

Both taxonomies are registered with `show_ui = false` — they are managed exclusively through the document editor meta box and the Settings panel, not through the default WordPress taxonomy screens.

---

## 4. Plugin Options — `wp_options`

One row per option. All keys are prefixed with `cirlot_docs_`.

| `option_name` | Type | Default | Description |
|---|---|---|---|
| `cirlot_docs_menu_name` | `string` | `Documents` | Admin sidebar menu label |
| `cirlot_docs_menu_icon` | `string` | `dashicons-media-document` | Dashicons class for the sidebar icon |
| `cirlot_docs_archive_slug` | `string` | `documents` | URL slug for the public archive |
| `cirlot_docs_default_audience` | `string` | *(empty)* | Default audience pre-selected in frontend shortcode |
| `cirlot_docs_default_type` | `string` | *(empty)* | Default document type pre-selected in frontend shortcode |
| `cirlot_docs_allowed_formats` | `array` (serialized) | `['pdf','word','excel']` | File formats allowed in the document editor |
| `cirlot_docs_gemini_api_key` | `string` | *(empty)* | Google Gemini API key |
| `cirlot_docs_gemini_model` | `string` | `gemini-2.5-flash` | Gemini model used for text generation |
| `cirlot_docs_audiences_list` | `string` | newline-separated defaults | One audience label per line. Used by `cirlot_docs_get_audiences()`. |
| `cirlot_docs_types_list` | `string` | newline-separated defaults | One document type per line. Used by `cirlot_docs_get_types()`. |
| `cirlot_docs_global_fields` | `string` (JSON) | `[{"id":"description","label":"Document Description","type":"textarea"}]` | JSON array of custom field definitions. Each object: `{ "id": string, "label": string, "type": "text"|"textarea"|"list" }` |

---

## 5. Transients — `wp_options` (with `_transient_` prefix)

| Key | TTL | Value | Description |
|---|---|---|---|
| `cirlot_docs_embed_model` | 1 hour | `{ "name": "models/text-embedding-004", "ver": "v1beta" }` | Cached result of the ListModels API call. Stores the first available Gemini embedding model and its API version. Avoids repeating the discovery request on every embedding call. |

---

## 6. WordPress Media Library — `wp_posts` + `wp_postmeta`

Uploaded document files are stored as standard WordPress attachments (`post_type = attachment`). The plugin stores only the attachment ID in `_document_file_id`. The actual file URL, path, and size are retrieved via standard WordPress functions (`wp_get_attachment_url`, `get_attached_file`).

## Summary Diagram

```
wp_posts
└── post_type = cirlot_document
    ├── post_title        (document title)
    └── post_id → wp_postmeta
                  ├── _document_file_id         → wp_posts (attachment)
                  ├── _document_pub_date
                  ├── _document_file_format
                  ├── _document_description
                  ├── _document_cf_{field_id}   (0..N custom fields)
                  ├── _document_summary
                  └── _document_embedding       (JSON float[768])

wp_term_relationships
└── post_id → document_audience terms (wp_terms)
└── post_id → document_type terms     (wp_terms)

wp_options
├── cirlot_docs_*          (plugin settings)
└── _transient_cirlot_docs_embed_model
```
